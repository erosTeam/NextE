# Plan Lifecycle

计划文件是任务证据，不是全局规范，也不产生工作、设备或远端操作授权。用户最新明确请求决定
当前任务。

## 目录

- `active/`：当前仍在推进或仍需明确闭环的计划。每份文件应有状态、范围、最近复核日期、验证和
  剩余事项。
- `completed/`：实现与声明的完成判据已经闭环的历史计划，只供追溯。
- `archive/`：已停用、被替代、长期未维护、仅供研究/证据或未完成但已不再活跃的材料。归档内容
  永不作为当前指令或授权。

## 维护规则

- 不建立全局 dispatch 队列，不从计划自动挑任务。
- 不把每次修复追加到永久 `active` 账本；一个任务一个计划，必要时链接当前源码/提交。
- 状态变化时移动文件，不靠在 `active` 文件顶部反复写 `historical`、`deprecated` 或 `complete`。
- 长期规则进入 `AGENTS.md` 或对应 `docs/agent-guides/`；协议真值进入架构/EH/同步等领域文档；
  单次日志、截图路径和旧设备信息只留在计划历史或 Git 历史。
- 归档前把仍然有效且尚未闭环的事项提炼到新的当前计划；无法确认是否仍有效时，标记归档原因，
  不把旧状态伪装成当前事实。
